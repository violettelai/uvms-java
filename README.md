# University Venue Management System (UVMS)

## Project Overview

This project was developed in **January 2023** using the **Java programming language**, covering both **frontend** and **backend** components.

## System Description

UVMS is designed to streamline venue booking within a university. It enables students to submit reservation requests and allows admins to manage and monitor all incoming submissions.

### User Roles

- **Student**
  - Log in with valid credentials
  - Add, edit, view, or delete their own reservation requests
  - View available and unavailable time slots for university venues

- **Admin**
  - Log in with admin credentials
  - View and manage all **pending** or **approved** reservation requests submitted by students
  - Access full reservation history across the system

## Key Features
- Role-based login for students and admins
- CRUD operations for reservation requests (Create, Read, Update, Delete)
- Venue availability tracking by date and time
- Admin dashboard for request management
- Simple, user-friendly interface for both users and admins

## Use Case
This system is ideal for any university environment where venue availability needs to be managed efficiently. It reduces manual handling of booking requests, avoids scheduling conflicts, and ensures smooth communication between students and administrators.


## Technologies Used
- **Language**: Java
- **Architecture**: Client-server (monolithic)
- **Database**: Local MySQL
